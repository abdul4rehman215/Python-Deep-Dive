1. What is a Python virtual environment?
   - An isolated Python environment with its own interpreter and packages.

2. Why are virtual environments important?
   - They prevent dependency and version conflicts between projects.

3. Which module is used to create virtual environments?
   - The built-in venv module.

4. What command creates a virtual environment?
   - python3 -m venv <env_name>

5. How do you activate a virtual environment on Linux?
   - source <env_name>/bin/activate

6. How can you tell a virtual environment is active?
   - The environment name appears in the shell prompt.

7. Why install packages inside a virtual environment?
   - To isolate project dependencies from the system Python.

8. Which command lists installed packages?
   - pip list

9. How do you exit a virtual environment?
   - Using the deactivate command.

10. Where are virtual environments used professionally?
    - CI/CD pipelines, DevOps workflows, microservices, and secure development.
